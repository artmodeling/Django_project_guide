1.	(Опционально. На выбор.) Создать папку, в которой будут храниться все проекты Django. Например, это будет папка «Django». 

2.	В папке «Django» создать папку с названием самого проекта. Далее будет выполняться создание интернет-магазина, поэтому папка будет названа «Online_store».

8.	Создать виртуальное окружение, последнее слово "venv" это имя этого виртульного окружения, имя можно дать любое, но виртуальное окружение принято называть «venv»:
```bash
python -m venv venv
```
9.	В проекте появилась папочка «venv»:  

<picture>
 <img alt="(007)" src="https://github.com/artmodeling/Django_project_guide/blob/main/Images/001_install_Django/007.png">
</picture>
 

10.	Перед работами обязательно нужно активировать глобальное окружение командой:

    1)	Если это Windows, необходимо выполнить эту команду. Поскольку в данном случае работы выполняются в операционной системе (ОС) Windows, нужно выбрать этот вариант:
    ```bash
    source venv/Scripts/activate
    ```

    2)	Если это Linux или Mac:
    ```bash
    source venv/bin/activate
    ```

    3)	Для справки – находится этот activate здесь (открывать этот файл не надо, делать с ним ничего не надо, эта информация только для справки):  
    <picture>
    <img alt="(008)" src="https://github.com/artmodeling/Django_project_guide/blob/main/Images/001_install_Django/008.png">
    </picture>
    
    **(Примечание.)** Сейчас эту команду выполнять **<ins>не нужно</ins>**, ее необходимо выполнить, когда виртуальное окружение нужно деактивировать, после выполнения этой команды надпись «(venv)» в терминале исчезнет:
    ```bash
    deactivate 
    ```

11.	В строке терминала после этой команды появилась надпись «venv», значит, виртуальное окружение активировалось, и можно продолжать работать, если эта надпись не появилась, то нужно выполнить все сначала и запустить (активировать) виртуальное окружение:

## Установка Django
 
12.	Теперь можно установить Django, чтобы он был именно в этом виртуальном окружении и не загромождал глобальное пространство:
```bash
pip install django
```

13.	Выполнить команду ниже еще раз – посмотреть, какие программы установлены (подсказка – если команда в этом терминале уже набиралась ранее, то ее можно найти, нажимая на клавиатуре на стрелочку вверх):
```bash
pip list
```

В этот раз результат будет уже другой:  

<picture>
 <img alt="(010)" src="https://github.com/artmodeling/Django_project_guide/blob/main/Images/001_install_Django/010.png">
</picture>

14.	(Если будет выполнено то, что написано в данном пункте, то после этого нужно будет снова активировать виртуальное окружение.) Если деактивировать виртуальное окружение и выполнить команду «`pip list`», то результат будет таким, каким он был прежде. Это подтверждает то, что программы были установлены в виртуальное окружение.

15.	Создание и заполнение файла «requirements.txt».

16.	Создать файл «requirements.txt». Создать его внутри самого проекта в данном случае в проекте «Online_store». (Убедиться, что файл создался именно в папке «Online_store», а не, например, в папке «venv».)

17.	(**Примечание.** На данном этапе этот пункт делать НЕ надо. Это нужно будет делать в будущем, если нужно будет этот проект восстановить на другом компьютере. Сейчас это только информация к сведению.) В этом файле указываем то, без чего этот проект не будет работать. Например, Django какой версии. Позже, когда нужно будет установить все, что указано в "requirements.txt", то нужно будет выполнить всего лишь следующую команду, а не устанавливать все необходимое руками:
```bash
pip install -r requirements.txt
``` 

18.	**(А это делать уже надо.)** Чтобы не заполнять этот файл руками, можно автоматизировать этот процесс и сохранить эти данные командой заморозки с указанием имени файла, куда это все нужно сохранить ("заморозить"):
```bash
pip freeze > requirements.txt
```

## Создание репозитория на гитхабе.

25.	Создать репозиторий на гитхабе:

Но на странице этого нового репозитория будут команды, которые пригодятся для работы с этим репозиторием дальше, поэтому их можно сохранить на компьютер в файл, они могут понадобиться позже:

26.	Теперь нужно связать этот репозиторий и локальный. Для этого необходимо выполнить следующие действия:

    1)	Сделать первый коммит (описание последовательности действий под изоражением):

    <picture>
    <img alt="(027)" src="https://github.com/artmodeling/Django_project_guide/blob/main/Images/001_install_Django/027.png">
    </picture>

       1. В VSC перейти в гит (слева иконка в виде разветвления), 

       2. выбрать файлы для коммита (на рисунке выше для коммита будут выбраны все файлы, это можно сделать для каждого файла отдельно, нажав на плюсик возле нужного файла), 

       3. написать текст коммита, например: «Init: Установил django, подготовил файлы README, requirements», 

    2)	нажать на кнопку «`Commit`»:

    Этот коммит создался локально на компьютере. Его нужно отправить на гитхаб. Для этого нужно связать локальный репозиторий с репозиторием на гитхабе. Для этого необходимо выполнить следующие шаги.

    3)	Связать репозитории :

        1.	Скопировать со страницы нового репозитория команду **(убедиться, что путь (ссылка), которая указана после слова «origin» начинается с «https»!!!)**, вместо «название_Вашего_репозитория» в Вашей команде будет указано название Вашего нового репозитория, который будет связан с локальным репозиторием и выполнить ее в терминале:
        ```bash
        git remote add origin https://github.com/название_Вашего_репозитория/Online_store.git
        ```

        2.	После этого репозитории будут связаны, но созданный коммит пока еще находится только локально на компьютере, на что указывает наличие только синей иконки «main» рядом с именем коммита.

        <picture>
        <img alt="(028)" src="https://github.com/artmodeling/Django_project_guide/blob/main/Images/001_install_Django/028.png">
        </picture>
        
        3.	Этот коммит нужно запушить в удаленный репозиторий. Команду для этого нужно так же скопировать со страницы нового репозитория и выполнить ее в терминале:
        ```bash
        git push -u origin main
        ```

        4.	Коммит запушился. Теперь он есть и локально на компьютере, и на удаленном репозитории. На локальном компьютере в VSC рядом с именем коммита появилось «облачко»:
